* **webpack's configuration file**
  * == JavaScript file or standard Node.js CommonJS module
    * / -- exports a -- webpack [configuration](configuration)
    * based upon its defined properties -> it's processed -- by -- webpack
  * allows
    * Import other files
      * -- via -- `require(...)`
    * utilities | npm
      * -- via -- `require(...)`
    * JavaScript control flow expressions
      * _Example:_ `?:`
    * constants or variables
    * write and execute functions
      * -- to generate a -- part of the configuration
  * **practices / should be avoided**
    * Access CLI arguments
      * better
        * write your own CLI or
        * [use `--env`](/api/cli/#env)
    * Export non-deterministic values
      * _Example:_ if you call webpack 2 -> SAME output files should be expected
    * Write long configurations
      * better
        * split the configuration | multiple files
  * there are MANY different ways to format and style your webpack configuration
    * -> stick with something consistent
  * check [Configuration section](/configuration/)
* _Examples:_

## Introductory Configuration

  **webpack.config.js**

  ```javascript
  const path = require('path');

  module.exports = {
    mode: 'development',
    entry: './foo.js',
    output: {
      path: path.resolve(__dirname, 'dist'),
      filename: 'foo.bundle.js',
    },
  };
  ```
